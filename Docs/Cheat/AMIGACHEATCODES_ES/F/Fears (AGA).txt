Cuando resolví el primer nivel y obtuve el código de nivel para el segundo nivel, 
Vi que este código era un número hexadecimal de 8 dígitos, pero pensé que era
no tenía sentido y el código era sólo una secuencia casual de dígitos...
¡No! ¡¡¡Eso está mal!!! En el nivel el código está codificado (bueno, es un código después de todo...:-))
todas las estadísticas importantes del jugador: su número de vidas, munición, nivel y el
tipo de armas.
De hecho, intenta terminar el primer nivel muchas veces, con diferente cantidad de munición o  
vidas, obtendrás un código diferente para el segundo nivel...
Tenga en cuenta que estos códigos no recuerdan cosas como energía, tesoros encontrados, tiempo.
usado, secretos encontrados, así que cuando ingresas a un nivel usando un código, todas estas estadísticas
estar en un nivel predeterminado.
Después de recopilar algunos códigos de acceso de Nivel 2, comencé a intentar comprender cómo funciona todo
estas informaciones están codificadas.
En primer lugar descubrí que la secuencia de 8 dígitos debe verse como la unión
de dos números de 4 dígitos: En el primero se codifica el nivel, el número de 
vidas y el tipo de armas disponibles, y en el segundo el número de los
Dos tipos de munición.
Ejemplo:
          código de nivel F8F856A5
                   \__/\__/
                    \ \__ significa 99 armas de tipo 1 y 28 de tipo 2
                     \_____ significa acceso al nivel 2 con arma y 0 vidas restantes
               
          código de nivel C9F545B9
                   \__/\__/
                    \ \__ significa 53 armas de tipo 1 y 14 de tipo 2
                     \_____ significa acceso al nivel 3 con pistola y ametralladora
                            y quedan 3 vidas
 
N.B: ¡¡¡los dos grupos de dígitos están completamente correlacionados!!! para que siempre puedas
     obtener los primeros 4 dígitos de un código de nivel y los últimos 4 de otro y
     ¡¡¡Tendrás otro código de nivel válido!!! 
     Ejemplo: usando los dos códigos anteriores puedes crear otros dos códigos:
     F8F845B9 y C9F556A5, ¡y esos también son códigos válidos!

| Explicación del PRIMER grupo de un código de nivel (los primeros 4 dígitos) |
------------------------------------------------------------------------

Aquí la cosa es un poco complicada: implica el uso de binario a hexadecimal.
conversiones y así si no sabes mucho al respecto... Salta a las conclusiones ;-)

En los primeros 4 dígitos están codificados el número de nivel, el tipo de armas y el 
número de vidas.
El número de código es un número hexadecimal, por lo que se puede escribir en binario:

 D8F5 -> 1101 1000 1111 0101

y el significado de cada grupo de bits es:

xxxx 100x x11x xxxxx
\__/ \____/ \__/|\_ =1 si tienes la ametralladora (0 en caso contrario)
 \ \ \ \__ =1 si tienes el lanzacohetes (0 en caso contrario)
  \ \ \____ en este triplete se codifica el nivel :
   \ \ 111 = nivel 1 110 = nivel 2 101 = nivel 3
    \ \ 100 = nivel 4 011 = nivel 5
     \ \_________ esto es una especie de suma de verificación para ver si el código es válido:
      \ No había descubierto cómo funciona esto pero puedes intentarlo.
       \ algunas combinaciones y usa la única que funcione
        \___________ en estos bits se codifica el número de vidas que tienes:
                           1111 = 0 vidas
                           1110 = 1 vida
                           1101 = 2 vidas
                           1100 = 3 vidas
                           ....... .......
                           0111 = 8 vidas
                           0110 = 9 vidas
